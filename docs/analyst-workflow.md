# MOEX CLI для финансового аналитика

## Рабочий процесс принятия решений

### 1. Анализ текущей позиции

#### Спецификация инструмента
```bash
./bin/moex security:specification SBER
```

**На что смотрю:**
- `ISIN` - уникальный идентификатор
- `List Level` - уровень листинга (1 = первый уровень, выше надёжность)
- `Type` - тип бумаги (акция обыкновенная/привилегированная)
- `Issue Size` - объём выпуска (оценка ликвидности)

**Решение:**
- List Level 1-2 → можно покупать
- List Level 3 → только для спекуляций
- Малый Issue Size → риск ликвидности

---

#### Рыночные данные
```bash
./bin/moex security:trade-data SBER
```

**На что смотрю:**
- `Last` - последняя цена
- `Open/High/Low` - диапазон дня
- `Volume Today` - объём торгов в деньгах

**Решение:**
- Объём выше среднего → интерес к бумаге
- Цена у верхней границы дня → бычье давление
- Цена у нижней границы → медвежье давление

---

#### Итоги торгов
```bash
./bin/moex security:aggregates SBER
```

**На что смотрю:**
- `Рынок акций` - основные торги
- `Рынок РЕПО` - спрос на кредитование под залог

**Решение:**
- Высокий объём РЕПО → бумаги в дефиците, ставки растут
- Низкий объём на акциях → уходят в РЕПО, ждать

---

### 2. Позиция в индексах

#### В какие индексы входит
```bash
./bin/moex security:indices SBER
```

**На что смотрю:**
- `IMOEX` - основной индекс, высокое влияние
- `MOEXBC` - голубые фишки, стабильность
- `MOEXFN` - финансовый сектор, секторальный риск

**Решение:**
- Входит в IMOEX → низкий риск ликвидности
- Только в секторальных → выше риск
- `Till` дата в прошлом → исключена из индекса, продавать

---

#### Вес в индексе (после реализации)
```bash
./bin/moex index:weight SBER IMOEX
```

**Интерпретация:**
- Вес > 10% → "тяжёлая" бумага, двигает индекс
- Вес 5-10% → значимая, но не критичная
- Вес < 5% → мало влияет на индекс

**Решение:**
- Тяжёлая бумага падает → весь индекс пойдёт вниз
- При коррекции индекса тяжёлые страдают больше

---

### 3. Сценарии использования

#### Сценарий 1: Оценка новой позиции

```bash
# Шаг 1: Базовая информация
./bin/moex security:specification LKOH

# Шаг 2: Текущая рыночная ситуация
./bin/moex security:trade-data LKOH

# Шаг 3: Позиция в индексах
./bin/moex security:indices LKOH

# Шаг 4: Ликвидность
./bin/moex security:aggregates LKOH
```

**Чек-лист:**
- [ ] List Level 1-2
- [ ] Входит в IMOEX
- [ ] Объём торгов > 1 млрд руб/день
- [ ] Вес в индексе > 3%

---

#### Сценарий 2: Ребалансировка портфеля

```bash
# Для каждой позиции в портфеле
for ticker in SBER GAZP LKOH GMKN NVTK; do
    echo "=== $ticker ==="
    ./bin/moex security:trade-data $ticker 2>/dev/null | grep -E "(Last|Volume)"
    ./bin/moex security:indices $ticker 2>/dev/null | grep "IMOEX"
done
```

**Анализ:**
| Тикер | Цена | Объём | В IMOEX | Вес в портфеле | Решение |
|-------|------|-------|---------|----------------|---------|
| SBER  | 318  | 3.0B  | ✅      | 15%            | HOLD    |
| GAZP  | 125  | 2.5B  | ✅      | 12%            | HOLD    |
| LKOH  | 7200 | 1.8B  | ✅      | 20%            | SELL 5% |
| GMKN  | 14500| 0.8B  | ✅      | 8%             | HOLD    |
| NVTK  | 1200 | 0.5B  | ✅      | 5%             | BUY 3%  |

---

#### Сценарий 3: Проверка перед продажей

```bash
./bin/moex security:aggregates SBER
```

**Смотрю:**
```
Рынок акций:
  Volume: 9,661,456  # ~3 млрд руб
  Trades: 57,804     # много сделок = ликвидность
  
Рынок РЕПО:
  Value: 51.5 млрд   # высокий = спрос на бумагу
```

**Решение:**
- Высокий объём → можно продавать без проскальзывания
- Высокий РЕПО → ставки высокие, кто-то занимает бумагу (возможно шорт)
- Низкий объём → лучше отложить продажу

---

### 4. Метрики для принятия решений

#### Ликвидность
```bash
./bin/moex security:aggregates SBER | grep "Рынок акций" -A5
```

| Объём в день | Ликвидность | Действие |
|--------------|-------------|----------|
| > 3 млрд     | Высокая     | Можно торговать крупно |
| 1-3 млрд     | Средняя     | Лимитные заявки |
| < 1 млрд     | Низкая      | Осторожно, проскальзывание |

---

#### Индексная значимость
```bash
./bin/moex security:indices SBER
```

| Количество индексов | Значимость |
|---------------------|------------|
| > 5                 | Высокая    |
| 3-5                 | Средняя    |
| 1-2                 | Низкая     |

### 5. Пример полного анализа

```bash
#!/bin/bash
# analyze.sh - полный анализ бумаги

TICKER=$1

echo "=== СПЕЦИФИКАЦИЯ ==="
./bin/moex security:specification $TICKER 2>/dev/null

echo ""
echo "=== РЫНОЧНЫЕ ДАННЫЕ ==="
./bin/moex security:trade-data $TICKER 2>/dev/null

echo ""
echo "=== ИНДЕКСЫ ==="
./bin/moex security:indices $TICKER 2>/dev/null

echo ""
echo "=== ЛИКВИДНОСТЬ ==="
./bin/moex security:aggregates $TICKER 2>/dev/null
```

**Запуск:**
```bash
./analyze.sh SBER > reports/SBER_$(date +%Y%m%d).txt
```

### 6. Интеграция с портфелем T-Invest

```bash
# Получить портфель из T-Invest
./bin/tinvest portfolio:analyze > portfolio.txt

# Для каждой позиции получить данные MOEX
cat portfolio.txt | grep -oE '[A-Z]{4}' | while read ticker; do
    echo "=== $ticker ===" >> moex_analysis.txt
    ./bin/moex security:indices $ticker >> moex_analysis.txt 2>/dev/null
done

```

### 7. Типовые решения

| Ситуация | Данные | Решение |
|----------|--------|---------|
| Бумага исключена из IMOEX | `Till` < сегодня | Продать |
| Объём торгов упал 3x | `Volume` низкий | Отложить покупку |
| Вес в индексе вырос | `weight` > 15% | Увеличить позицию |
| Высокий РЕПО | `repo value` высокий | Краткосрочная продажа |

## Команды справочника

| Команда | Назначение | Частота использования |
|---------|------------|----------------------|
| `security:specification` | Базовая информация | При первой покупке |
| `security:trade-data` | Текущие цены | Ежедневно |
| `security:aggregates` | Итоги торгов | Еженедельно |
| `security:indices` | Индексы | При ребалансировке |
| `index:weight` | Вес в индексе | При ребалансировке |
| `index:composition` | Состав индекса | При подборе бумаг |
| `candles` | История цен | Для теханализа |
